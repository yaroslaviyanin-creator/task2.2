Сборка и тестирование утилиты шестнадцатеричного вывода (task2.2)

1. ГЕНЕРАЦИЯ ПРОЕКТА
Создает нужные файлы для сборки в отдельной папке build. 
Выполнять из корневой директории проекта:
> cmake -B build

(Опционально для конкретной версии студии: cmake -B build -G "Visual Studio 17 2022" -A x64)
Проект можно открыть в Visual Studio через файл CMakeLists.txt или сгенерированный .sln.

2. СБОРКА ПРОЕКТА
Компиляция исходного кода. Выполнять из корня проекта:
> cmake --build build --config Debug
(Для релизной версии замените Debug на Release)

3. ЗАПУСК ПРОГРАММЫ (ПРИМЕРЫ)
После успешной сборки исполняемый файл находится в папке build\app\Debug\
> .\build\app\Debug\my_program.exe -i test_file.bin
> .\build\app\Debug\my_program.exe -i test_file.bin -g 4 -n 2 -o 4 -l 8

4. ЗАПУСК АВТОМАТИЧЕСКИХ ТЕСТОВ (Unity + CTest)
ВАЖНО: Для корректной работы тестов необходимо сначала перейти в папку build!
> cd build
> ctest -C Debug --output-on-failure

---
ПОЛЕЗНЫЕ ЗАМЕТКИ:
- При переносе проекта на другой компьютер ОБЯЗАТЕЛЬНО удалите папку "build", так как в ней сохраняются абсолютные пути старого ПК. После удаления повторите шаги 1 и 2.
- Флаг --output-on-failure при запуске тестов позволяет увидеть подробный лог (с номерами строк), если какой-то из тестов упадет.
